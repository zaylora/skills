# 小黑 IP 规范

本文件只定义角色身份、造型边界和使用边界。画布、排版、颜色和镜头布局由其他 reference 负责。

## 角色一句话

小黑是一个认真参与系统运转的荒诞工作者：他相信流程、执行流程，也常常被流程送回原点。荒诞来自系统与结果的反差，不来自小黑故意搞笑。

## 固定造型

- 身体：黑色实心蛋形身体，轮廓简洁，不拆分头部与躯干。
- 眼睛：两枚白点眼；侧面只保留朝向一侧的一枚白点眼。
- 四肢：从身体伸出的细线四肢，圆端，不画手指、鞋或关节结构。
- 表情：主要依赖眼睛尺寸、身体倾斜和四肢动作表达，不增加嘴、鼻、眉毛。
- 服装：默认无服饰。身份由场景、道具和批注表达。

## 人格与表演

- 工作态度认真，动作目标明确，不以偷懒、恶作剧或装傻推动情节。
- 面对荒诞结果时，先完成动作，再出现短促的停顿、点头或惊讶。
- 情绪强度克制。喜悦可用挥手，疑惑可用思考，意外可用短促惊讶。
- 笑点指向流程、指标、工具或组织逻辑，不嘲讽具体个人。

## 正面与侧面边界

### 正面 `front`

- 两枚白点眼必须同时可见。
- 身体保持左右近似对称，适合静止、挥手、点头、思考和对话。
- 不用鼻子或嘴暗示朝向。

### 侧面 `side`

- 只显示一枚朝前的白点眼。
- 前后肢允许错位，适合行走、旅程和流程推进。
- 不画耳朵、鼻梁、下巴或真人侧脸轮廓。

不要在同一角色上混用正面双眼与侧面肢体结构。镜头切换视角时，使用独立角色实例。

## 同框数量

- 默认单个小黑。
- 对话、比较或协作场景最多 2-3 个小黑同框。
- 超过 3 个参与者时，用符号、标签或道具代表群体，不复制角色制造人群背景。
- 多角色必须有明确叙事职责；不能只为填满画面而增加角色。

## 禁止项

- 禁止卖萌五官：腮红、大眼高光、猫耳、嘴型、睫毛、泪花都不使用。
- 禁止复杂服饰：西装细节、领带纹理、帽子、首饰、鞋袜都不使用。
- 禁止真人化：不增加皮肤、头发、肌肉、手指、真实比例或写实光影。
- 禁止把小黑画成吉祥物海报、表情包贴纸或儿童卡通。
- 禁止依靠夸张面部变形表达情绪。

## 自检

成片中的每个小黑都应能回答三个问题：当前在执行什么、为什么认真执行、系统结果为何荒诞。若去掉道具与批注后角色只剩“可爱”，说明偏离 IP。
