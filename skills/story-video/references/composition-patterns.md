# 镜头构图 Patterns

这 8 个 ID 是 `story.json` 的镜头布局 schema，不是额外框架。它们只帮助 Agent 把叙事意图映射到稳定构图；实现仍是同一个 HTML、CSS 与 GSAP timeline。

## 目录

- `character-state`：角色状态
- `dialogue`：对话
- `journey`：旅程
- `process`：流程
- `comparison`：比较
- `cause-effect`：因果
- `reveal`：揭示
- `summary`：总结

## 共同规则

- 每镜只选一个 pattern，并让该 pattern 的关系成为唯一视觉重点。
- 所有 pattern 都使用白底、大量留白与五层结构。
- 优先复用小黑、32 个 props、气泡与三色批注，不引入网络图片。
- pattern 决定布局，不决定故事内容、角色人格或颜色。

## Pattern: `character-state`

**适用场景**

- 介绍小黑当前在做什么、等待什么或意识到什么。
- 一个动作或情绪就是该镜头的主要信息。

**核心布局**

- 小黑位于画面中部偏左或正中，保留完整轮廓。
- 单个关键道具放在动作方向上，气泡位于眼睛朝向一侧。
- 批注只标记状态变化，不重复字幕。

**优先资产**

- `xiaohei.spawn`；动作优先 `idle`、`wave`、`nod`、`think`、`surprise`。
- 道具优先 `laptop`、`coffee`、`docs`、`question`、`excl`。

## Pattern: `dialogue`

**适用场景**

- 两个角色交换一句关键信息，或同一角色面对系统提示。
- 对话内容比空间移动更重要。

**核心布局**

- 两个小黑左右对置，最多 2-3 个同框。
- 每次只突出一个气泡，避免气泡交叉或同时抢读。
- 中央留空作为对话张力区，字幕放在下方独立层。

**优先资产**

- 正面或侧面小黑组合；动作优先 `idle`、`wave`、`nod`。
- 气泡、`phone`、`mailbox`、`question`、`check`。

## Pattern: `journey`

**适用场景**

- 展示小黑从一个节点前往另一个节点。
- 路径、阻碍或返回原点是叙事重点。

**核心布局**

- 路径沿水平方向展开，起点与终点不超过三个节点。
- 侧面小黑沿路径运动，camera 可做短距离平移。
- 箭头只标方向，不能形成复杂流程图。

**优先资产**

- `xiaohei.spawnSide` 与 `walk`。
- `arrow`、`gate`、`ladder`、`pipe`、`mailbox`、`up`。

## Pattern: `process`

**适用场景**

- 解释输入如何经过 2-4 个步骤得到结果。
- 系统运转方式比单个角色状态更重要。

**核心布局**

- 步骤从左到右单线排列，当前步骤用唯一批注色标记。
- 小黑站在当前操作节点附近，不能覆盖连接线。
- 若步骤超过四个，拆成多个镜头，不缩小到流程海报。

**优先资产**

- `conveyor`、`funnel`、`gate`、`blackbox`、`pipe`、`gear`。
- `arrow`、`check`、`cross` 只表达状态，不增加分支层级。

## Pattern: `comparison`

**适用场景**

- 对比两种路径、前后状态或承诺与结果。
- 观众需要同时扫描两侧差异。

**核心布局**

- 画布分为左右两列，但不使用卡片容器。
- 两侧使用相同尺度与基线，中间保留明确间隔。
- 批注指出一个关键差异，字幕给出结论。

**优先资产**

- 两个同视角小黑、`balance`、`scale`、`check`、`cross`。
- 重复道具应使用相同尺寸，避免视觉偏置。

## Pattern: `cause-effect`

**适用场景**

- 展示一个动作触发一个意外或荒诞结果。
- 需要清晰回答“因为 A，所以 B”。

**核心布局**

- 原因在左、结果在右，中间使用单向箭头或连线。
- 结果区域可以比原因稍大，但不能变成双重主角。
- 小黑位于原因端，反应动作发生在结果出现之后。

**优先资产**

- `arrow`、`boom`、`excl`、`sweat`、`cross`、`loop`。
- 动作优先 `wave`、`nod`、`surprise`。

## Pattern: `reveal`

**适用场景**

- 先建立预期，再揭示隐藏结果、真实标签或黑箱输出。
- 揭示是该镜头唯一高潮。

**核心布局**

- 被揭示对象位于中心，揭示前后位置保持稳定。
- 使用遮挡物移开、批注出现或 camera 轻推完成揭示。
- 不用渐变、闪光或网络图片制造悬念。

**优先资产**

- `blackbox`、`docs`、`mailbox`、`question`、`excl`、`boom`。
- 小黑动作优先 `think`、`surprise`。

## Pattern: `summary`

**适用场景**

- 收束故事，重申一个结论或留下一个反问。
- 用于最后镜头，不继续引入新事实。

**核心布局**

- 标题或结论居中，小黑与一个象征性道具形成小尺度支点。
- 最多保留三个短关键词，不复刻前面所有镜头。
- 末尾可整体淡出，但仍需先有完整入口与阅读停留。

**优先资产**

- `check`、`cross`、`loop`、`balance`、`idea`、`question`。
- 小黑动作优先 `idle`、`nod`、`wave`。

## 选择检查

若两个 pattern 都看似适用，选择能用更少对象表达关系的一个。需要同时画流程、比较和总结时，应拆镜，而不是组合出新的 pattern ID。
