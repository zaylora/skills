# RAG 科普短视频 · 设计文档

> 用一条 1–2 分钟的竖屏口播视频,给完全不懂技术的普通人讲清楚 RAG 到底怎么工作。
> 制作工具:`chengfeng-videocut-skills`(剪口播 + 口播成片)。

日期:2026-07-16

---

## 1. 项目概述

| 项 | 设定 |
| --- | --- |
| 主题 | 讲清楚 RAG 到底是怎么工作的 |
| 目标 | 看完能明白:RAG = 让 AI「开卷考试」,答题前先翻资料再回答 |
| 受众 | 完全不懂技术的普通人(全程生活比喻,不出现代码) |
| 时长 | 1–2 分钟(口播稿约 400 字,念读约 1.5 分钟) |
| 画幅 | 3:4 竖屏(逻辑画布 1080×1440,导出 1620×2160) |
| 形式 | 纯口播配音 + 小黑风格动画画面(不出镜、不录屏) |
| 动画风格 | `xiaohei`(小黑漫画感 SVG 动效,skill 默认) |
| 发布平台 | 小红书 / 视频号 / 抖音 / 快手等竖屏平台 |

## 2. 核心创意

**核心比喻:开卷考试。**

- 普通 AI 答题 = 闭卷考试:靠训练时「背」下来的记忆,记混或没背过就连蒙带猜 → 这就是「AI 幻觉」。
- RAG = 开卷考试:允许它答题前先翻资料,找到最相关的那几页,照着回答。

**一句话主张(全片落点):**

> RAG 并没有把 AI 变得更聪明,它只是把闭卷考试,变成了开卷考试。

## 3. 分镜脚本(6 镜)

### 镜头 1 · 钩子(约 0–12s)
- 口播:"你有没有发现,AI 有时候特别会一本正经地胡说八道?你问它一件真实的事,它张口就编一个,还说得跟真的一样。这到底是为什么?今天用一个你一定经历过的场景,一分钟讲明白。"
- 画面:自信的 AI 头像在说话,旁边「啪」地弹出红色标签「❌ 编的」,手绘问号乱飞。

### 镜头 2 · 闭卷考试 = 幻觉根源(约 12–30s)
- 口播:"其实普通 AI 答题,靠的是它训练时'背'下来的记忆,就像一场闭卷考试。背得牢的能答对;可一旦碰到它没背过、或者记混的,它又不肯交白卷,于是只能连蒙带猜——这就是我们常说的'AI 幻觉'。"
- 画面:考场里 AI 坐课桌前,头顶「闭卷」牌子;一道题答不出、冒汗,最后瞎填一个答案被打叉。

### 镜头 3 · RAG 登场 = 开卷考试(约 30–42s)
- 口播:"那怎么办?办法很简单——让它改成开卷考试。这个方法就叫 RAG。说白了,就是不让 AI 光靠脑子里的记忆硬答,而是允许它一边翻资料、一边答。"
- 画面:「闭卷」牌子翻转成「开卷」;AI 桌上出现一摞书和一个资料库图标,表情轻松下来。

### 镜头 4 · 三步流程(约 42–78s)——核心
- 口播:"它分三步。第一步,你把问题递过去,比如'我们公司的请假流程是什么'。第二步,系统先不急着答,而是拿着这个问题,去一个专门的资料库里,飞快翻出最相关的那几页——注意,不是整本书,就那几页最有用的。第三步,把这几页资料和你的问题一起交给 AI,等于告诉它:别猜,照着这几页说。"
- 画面:三步依次点亮。① 用户递出问题卡片;② 一个「图书管理员」角色冲进书库,抽出高亮的几页、其余变灰;③ 这几页 + 问题一起塞给 AI,AI 照着念。

### 镜头 5 · 好处 + 结论(约 78–95s)
- 口播:"这么一来好处就出来了:答案有据可查,能用上最新的、甚至你自己私有的资料,也不容易瞎编了。所以你看,RAG 没有把 AI 变得更聪明,它只是把闭卷考试,变成了开卷考试。"
- 画面:三个对勾依次跳出「有据可查 / 最新+私有资料 / 少瞎编」;最后闭卷 vs 开卷两画面并排,开卷这边盖大对勾。

### 镜头 6 · 收尾(约 95–105s)
- 口播:"看懂了点个赞,下次讲讲它是怎么'翻到对的那几页'的。"
- 画面:结束卡,大字「RAG = 让 AI 开卷考试」。

## 5. 录制指导

因为是纯配音,画面之后全部换成动画,所以**录制时画面拍什么都无所谓**,关键是把口播录清楚。

- **设备**:手机或电脑都行。手机对着桌子/墙录一段视频即可(要有画面才是 mp4,方便直接进剪口播);纯音频也可以,后期我用 ffmpeg 配黑底转成 mp4。
- **环境**:安静、无回声,离麦克风近一点。
- **念法**:
  - 语速自然,句与句之间**留短暂停顿**(剪口播按静音自动分句,停顿能提高切分准确度)。
  - **念错不要停、不要重录整条**:直接把念错的那一句从头再念一遍。剪口播的「删前保后」会自动删掉前面念错的版本、保留后面对的版本。
  - 专有名词「RAG」按字母念「R-A-G」或「rag」,保持全片一致,方便字幕校对。
- **输出**:保存成一个视频文件(`.mp4` / `.mov`),放进项目目录,命名建议 `rag_raw.mp4`。

## 6. 后续制作流程(用 chengfeng-videocut-skills)

```
录好的 rag_raw.mp4
   ↓  剪口播(需先配火山 API Key)
source_cut.mp4 + subtitles.srt
   ↓  口播成片(3:4 竖屏,xiaohei 动画)
分镜稿 HTML → 时间线预览 → 导出 1620×2160 MP4
   ↓  验收(ffprobe + 抽关键帧)
最终成片
```

1. **剪口播**:转录 → 识别口误/停顿/重复 → 生成审核网页 → 你在网页确认删哪里 → 粗剪 → 基于剪后视频重转写 + AI 校对字幕。产出 `source_cut.mp4` 和 `subtitles.srt`。
2. **口播成片**:按本文档的 6 镜分镜,先生成 HTML 分镜核对页给你确认(**不直接导出**),再做时间线预览,最后合成 3:4 竖屏 MP4。动画用小黑风格,考场/书库/图书管理员等元素按分镜实现。
3. **验收**:用 ffprobe 检查分辨率 1620×2160、时长与音频;抽关键帧核对每一镜画面正确。

## 8. 前提与风险

- **火山引擎 API Key 未配置**:剪口播的转录依赖「录音文件识别 2.0」。需要先到火山控制台开通并创建 Key,填入 `.claude/skills/chengfeng-videocut-skills/.env` 的 `VOLCENGINE_API_KEY`。开通地址:https://console.volcengine.com/speech/new/setting/activate?projectName=default
- **运行环境为 Windows**:该 skill 的脚本以 bash / node 为主,并使用 `ln -sf` 建符号链接。node 脚本、ffmpeg 跨平台可用;符号链接在 Windows Git Bash 下可能失败,届时改用文件复制代替即可,不影响产物。
- **审核硬规则**:剪口播到审核网页必须停下交给用户确认,Agent 不擅自剪辑;口播成片必须先给分镜稿和时间线预览确认,不跳过直接导出。
