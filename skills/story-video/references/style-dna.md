# 小黑故事视频 Style DNA

本文件只定义所有镜头共享的视觉常量。角色人格见 `xiaohei-ip.md`，镜头布局见 `composition-patterns.md`。

## 画布

- 固定横屏 16:9，尺寸为 1920x1080。
- 背景固定白色 `#ffffff`。
- 主体、正文、角色与道具线条统一使用 `#1a1a1a`。
- 画面保持大量留白，单镜单意；每镜只让一个关系或结论成为视觉中心。

## 颜色

主体色只有 `#1a1a1a`。批注与强调色只允许以下三种：

| 色值 | 角色 |
| --- | --- |
| `#e2483d` | 错误、阻断、冲突 |
| `#f5a623` | 等待、疑问、风险 |
| `#2f7dd1` | 路径、说明、确认 |

强调色不能用于大面积背景、角色身体或装饰填充。一个镜头优先只使用一种批注色；确有对照时最多两种。

## 线条

- 常规线宽约 `2.2px`，线帽与连接点使用圆端。
- 允许轻微手绘偏移，但必须是确定性抖动：相同 seed、相同输入必须得到相同路径。
- 角色四肢、道具、箭头和批注使用同一线性语言。
- 不叠加粗描边、荧光描边或多层轮廓。

## 排版

- 使用中文系统字体组合：`"Microsoft YaHei", "PingFang SC", "Hiragino Sans GB", sans-serif`。
- 标题字号不小于 `60px`，正文与气泡文字字号不小于 `20px`。
- 批注和说明也按视频阅读距离设计，不使用网页级小字。
- 所有文字固定 `letter-spacing: 0`，不随 viewport 缩放字号。
- 标题用高字重，正文用常规字重；不再引入第二套展示字体。
- 文本靠容器宽度自然换行，不使用 `<br>` 强制断行。

## 空间与层级

- 主叙事对象占画面的中部或三分线交点，不铺满全屏。
- 角色、道具、气泡、批注之间保留可辨认的空隙。
- 字幕与主动作分区，不能压住角色、道具或批注。
- 使用尺寸、位置与出现顺序建立层级，不用视觉特效制造层级。
- 任何固定格式元素都给出稳定尺寸，动画不能改变布局占位。

## 动效

- 先用静态 CSS 完成每个镜头的 hero frame，再用 `tl.from()` 添加入口。
- 首个动画在 `0.1-0.3s` 之间开始。
- 位移短而明确，优先使用 `x`、`y`、`scale`、`rotation` 和 `opacity`。
- 循环必须有限；所有动作可在任意时间点确定性 seek。
- 非最后镜头不额外退场，场景 transition 本身负责退出。

## 禁止项

- 禁止渐变，包括背景渐变、文字渐变和装饰渐变。
- 禁止阴影，包括盒阴影、文字阴影和 SVG 投影。
- 禁止纸纹、噪点底纹、胶片颗粒和仿旧材质。
- 禁止装饰卡片化：页面分区不能变成一组悬浮卡片。
- 禁止网络图片、图库素材、远程照片和外链插画。
- 禁止用大色块、光晕、球体或无叙事意义的几何装饰填空。

## 一致性检查

暂停在任意 hero frame：应先看到一个清楚的动作或关系，再读到一句说明；白底仍是画面主体，强调色只承担批注意义。若去掉动画后布局不成立，应先修静态 CSS，而不是补更多 tween。
